# Usage Policy

> 사내 배포 시 온보딩 자료로 활용한다.

## 1. 접속 및 로그인

### 현재 (데모 버전)
- 이메일 + 이름 입력 → "시작하기" 버튼
- 동일 이메일로 로그인하면 기존 데이터가 유지된다
- 별도 비밀번호 없음 — 이메일이 사용자 식별자

### 운영 버전 (v2 계획)
- Google Workspace 계정으로 SSO 로그인 전환 예정
- `@saladlab.com` 계정만 접근 가능

### 세션 관리
- 로그인 세션은 브라우저 localStorage에 저장
- Access Token은 15분마다 자동 갱신 (앱 사용 중에는 자동 처리)
- 브라우저 탭을 오래 닫아 두면 재로그인이 필요할 수 있음

## 2. 할일 관리

### 생성
- 오른쪽 상단 **"+ 새 할일"** 버튼 또는 키보드 `n` 키
- 제목은 필수 (최대 200자), 나머지 항목은 선택
- 우선순위를 지정하지 않으면 **MEDIUM(🟡)** 으로 자동 설정
- 저장: **저장 버튼** 또는 `Enter` / 취소: `Esc`

### 수정
- **제목**: 제목 텍스트 클릭 → 직접 수정 → `Enter` 저장 / `Esc` 취소
- **우선순위**: 🔴🟡🟢 아이콘 클릭 → 드롭다운에서 선택 → 즉시 저장
- **상태**: 체크박스 클릭 (TODO ↔ DONE) 또는 **"작업 시작"** 버튼 (TODO → IN_PROGRESS)

### 상태 의미

| 상태 | 의미 | 색상 |
|------|------|------|
| TODO | 아직 시작 안 함 (기본값) | 회색 |
| IN_PROGRESS | 현재 진행 중 | 파란색 |
| DONE | 완료 | 초록색 |

- DONE으로 변경하면 취소선 애니메이션이 적용됨
- DONE → TODO로 되돌리기 가능 (완료 취소)

### 삭제
- 카드 우측 🗑️ 아이콘 클릭 → 확인 다이얼로그 → **"삭제"** 클릭
- 삭제된 항목은 **휴지통**으로 이동 (영구 삭제 아님)
- 실수로 삭제했다면 왼쪽 메뉴의 **"휴지통"** 에서 복원 가능

## 3. 필터 사용법

- 목록 상단의 필터 바에서 **상태 / 우선순위 / 카테고리** 기준으로 필터링
- 여러 조건을 동시에 선택하면 **모든 조건을 충족하는** 항목만 표시 (AND 조건)
- "Clear filters" 버튼으로 전체 목록으로 복귀

**예시**:
- 우선순위 HIGH + 상태 TODO → "아직 시작 안 한 긴급 할일"만 표시
- 카테고리 "업무" + 상태 IN_PROGRESS → "진행 중인 업무 할일"만 표시

---

## 4. 카테고리 & 태그

### 카테고리
- 나만의 폴더 개념 — 계정당 최대 **20개**
- 할일 생성/수정 시 카테고리 지정 가능
- 필터 바에서 카테고리 단위 필터링 가능
- 색상 지정으로 시각적 구분

### 태그
- 여러 할일에 공통으로 붙이는 라벨 개념
- 할일당 최대 **10개** 태그 지정
- 태그는 모든 사용자가 공유 (전역 태그)
- 현재 태그 단독 필터 미지원 — v2에서 추가 예정

---

## 5. 리마인더

- 할일에 **마감일**을 설정하면 자동으로 마감 **1시간 전** 리마인더가 생성됨
- 현재는 서버 콘솔 로그로만 출력됨 (실 이메일 발송은 v2에서 구현)
- 할일을 DONE으로 완료하면 리마인더가 자동으로 취소됨
- 완료 후 다시 TODO로 되돌리면 마감일이 아직 남아 있을 경우 리마인더가 재생성됨

---

## 6. 휴지통

- 삭제된 할일은 **30일 동안** 휴지통에 보관됨
- 휴지통에서:
  - **복원**: 원래 목록으로 되돌아감, 모든 데이터 유지
  - **영구 삭제**: 즉시 완전 삭제, 복원 불가
- 30일이 지나면 자동으로 영구 삭제됨 (복원 불가)

## 7. 키보드 단축키

| 단축키 | 동작 | 조건 |
|--------|------|------|
| `n` | 새 할일 생성 창 열기 | 입력 필드에 포커스가 없을 때 |
| `Enter` | 인라인 편집 저장 / 폼 제출 | 편집/입력 중일 때 |
| `Esc` | 편집 취소 / 모달 닫기 | 편집/모달 열려 있을 때 |
| `Tab` | 폼 필드 간 이동 | 생성/편집 폼 안 |

## 8. 데이터 관리 정책

### 데이터 소유권
- 각 사용자는 본인이 생성한 할일, 카테고리만 조회·수정·삭제 가능
- 다른 사용자의 데이터에는 접근 불가

### 데이터 보존
| 데이터 | 보존 기간 | 삭제 방식 |
|--------|-----------|-----------|
| 정상 할일 | 무기한 | 사용자가 직접 삭제 |
| 삭제된 할일 (휴지통) | 30일 | 자동 영구 삭제 |
| 발송된 리마인더 | 무기한 (로그용) | 별도 삭제 정책 없음 |
| 사용자 계정 | 무기한 | 탈퇴 기능 미구현 (v2 예정) |

### 현재 제한사항 (v1)
- 데이터 내보내기(Export) 기능 없음
- 계정 삭제 기능 없음
- 여러 기기에서 동시 사용 시 실시간 동기화 없음 (새로고침 필요)
- 오프라인 사용 불가 (인터넷 연결 필수)

## 10. 알려진 제한사항 & 예정 개선
